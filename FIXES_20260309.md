# Fixes applied on 2026-03-09

## What was fixed

### 1) Per-agent model selection from CLI
Added support for assigning models independently to pipeline agents.

New CLI flags:
- `--agent-models 'planner=gpt-4;coder=local:Qwen/demo;fixer=gpt-4o-mini'`
- `--planner-models ...`
- `--coder-models ...`
- `--fixer-models ...`

These flags are available in both:
- `pipeline_cli.py generate-solver`
- `pipeline_cli.py run`

The routing is propagated into `AgentLaboratory/perm_pipeline/run_perm_pipeline.py`.

### 2) More stable solver generation
To reduce unnecessary fallback to the offline baseline, the generation flow now includes:
- stricter model response parsing;
- automatic one-shot format-repair retry when the coder response does not contain a valid Python file;
- dedicated fixer-model routing;
- baseline patch attempt before final offline fallback.

This makes the following failure mode much less likely:

```text
[!] Failed to generate a locally validated solver with the configured models.
```

### 3) Kaggle authentication fix
Kaggle auth handling now supports both common credential formats:
- legacy `kaggle.json` (`username` + `key`)
- token-based auth (`access_token` / `api_token` / plain token file)

The CLI submit path now builds the correct environment for both modes.

## Recommended examples

### Generate solver with different models per agent
```bash
python3 pipeline_cli.py generate-solver \
  --competition cayleypy-rapapport-m2 \
  --out generated/solver.py \
  --agent-models 'planner=gpt-4;coder=local:Qwen/demo;fixer=gpt-4o-mini'
```

### Full run with dedicated planner/coder/fixer models
```bash
python3 pipeline_cli.py run \
  --competition cayleypy-rapapport-m2 \
  --output submissions/submission.csv \
  --planner-models gpt-4 \
  --coder-models local:Qwen/demo \
  --fixer-models gpt-4o-mini
```
