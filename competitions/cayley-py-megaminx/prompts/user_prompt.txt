We are solving the Kaggle competition "CayleyPy Megaminx Solve Optimally".
Reference: https://www.kaggle.com/competitions/cayley-py-megaminx

PROBLEM SUMMARY
- State: a permutation vector of length 120.
- Goal: transform the given state into the central solved state from competitions/cayley-py-megaminx/data/puzzle_info.json.
- Legal moves are the generator names from puzzle_info.json.
- Official move notation used by the competition: U, D, F, B, L, DR, BL, FR, BR, FL, R, DL and their inverses.
- Kaggle submissions use dot-separated move names in the path column.

INPUT / OUTPUT CONTRACT
- Implement solve(vec) -> (moves, sorted_array)
- vec is a Python list[int]
- moves must be a list[str] with legal generator names from puzzle_info.json.
- sorted_array must be the final state after applying moves
- Script mode must print JSON: {"moves": ..., "sorted_array": ...}

ALGORITHM REQUIREMENTS
- The code must run in polynomial time.
- Do NOT use BFS, DFS, A*, IDA*, beam search over growing depth, or whole-state-space enumeration.
- Constant-depth lookahead over the fixed generator set (depth 1 or 2) is allowed, because that remains polynomial.
- Do NOT invent move names; use only keys from puzzle_info.json.

RECOMMENDED IMPLEMENTATION ORDER
1. DATA-AWARE EXACT BASELINE
   If competitions/cayley-py-megaminx/data/test.csv and sample_submission.csv exist, build a dictionary:
       initial_state string -> known valid path
   Return that exact path for exact matches. This is deterministic, valid for the bundled dataset, and polynomial-time.
2. GENERIC FALLBACK FOR UNSEEN STATES
   - Load central_state and generators from puzzle_info.json.
   - Implement apply_perm(state, perm) and apply_moves(state, moves).
   - Score a state with a polynomial-time objective such as:
       (a) number of positions already equal to central_state (maximize),
       (b) longest correct prefix (maximize),
       (c) total mismatches (minimize).
   - Evaluate every legal 1-move candidate, and optionally every legal 2-move macro.
   - Repeatedly choose the move or macro that improves the score the most.
   - Stop when central_state is reached.
   - If bundled lookup files are present, prefer them over any heuristic fallback so every emitted path is competition-valid.

IMPLEMENTATION DETAILS
- Keep the code self-contained; use only the standard library.
- It is fine to work on a copy of vec.
- Always verify that sorted_array equals the state produced by the emitted moves.
- Return exactly one complete Python file inside one ```python``` block.
- Do not add explanations outside the code block.
