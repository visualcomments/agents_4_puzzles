We are solving the CayleyPy RapaportM2 Kaggle competition.
Reference: https://www.kaggle.com/competitions/cayleypy-rapapport-m2

TASK
Write solve_module.py that sorts a permutation using ONLY the legal moves I, S, K.

LEGAL MOVES
- I: swap positions 0 and 1
- S: swap (0,1), (2,3), (4,5), ... simultaneously
- K: swap (1,2), (3,4), (5,6), ... simultaneously

INPUT / OUTPUT CONTRACT
- Implement solve(vec) -> (moves, sorted_array)
- moves must be list[str] with every token in {"I","S","K"}, or the string "UNSOLVED"
- sorted_array must be the final array after applying the moves
- Script mode must print JSON: {"moves": ..., "sorted_array": ...}

ALGORITHM REQUIREMENTS
- Polynomial time only.
- No BFS / DFS / A* / exhaustive search.
- No built-in sorting for the core algorithm.
- Do not emit any move letters other than I, S, K.

RECOMMENDED IMPLEMENTATION ORDER
1. If bundled test.csv + sample_submission.csv exist, you may first build an exact lookup baseline:
       permutation string -> valid path
   Return that exact path for exact matches.
2. Otherwise implement the constructive cycle-order algorithm:
   - Let C = S then K.
   - The index order of C is the cycle [0,2,4,...,largest even, largest odd,...,3,1].
   - Using conjugation, swap(cycle[j-1], cycle[j]) = P + [I] + reverse(P), where P = C^{-j} and C^{-1} = K then S.
   - Perform insertion sort along this cycle order.
3. This gives a fully constructive polynomial-time solver.

IMPLEMENTATION DETAILS
- Work on a copy of vec.
- Implement apply_I / apply_S / apply_K in place.
- Verify that sorted_array is exactly the result of the returned moves.
- Return exactly one complete Python file inside one ```python``` block.
- Do not add explanations outside the code block.
